📐 文档体系方法论

用 HTML 单文件当 PRD · 7 原则 · 6 层结构 · 可抄代码片段
这份是「BeeX 用 HTML 文件当 PRD」这套做法的方法论提炼,目的是让另一个项目照着同样思路从 0 搭一套。
用 BeeX 当例子,但规则都是通用的——换项目时把业务词换掉、把配色 token 换掉即可。

0. 一句话

把 PRD 拆成「一页总纲 + N 个场景流程图 + 若干设计方案 + 一张全盘状态表」,
每篇是一个零依赖的 HTML 单文件,内联一套设计系统,流程图用本地 Mermaid(文本即图),
全程无黑话、单一真值来源、统一状态徽章,file:// 双击即看、git 管理、零构建。

1. 七条核心原则(为什么这么做)

原则含义落地动作
① 无黑话任何术语,团长/运营/新人/测试都能看懂维护一个「禁用词表」(BeeX 禁:双轨/Mentor/cascade),review 时扫
② 一页读懂优先先有一篇「业务总纲」一页讲清整个生意怎么转,再深入overview.html 永远是第一篇,导航里用大卡片置顶
③ 单一真值来源数值/状态只在一个地方定义,其它文档指向它而非复制每页页脚写「数值一律以 X(线上/设计稿)为准」;不在多篇里抄同一个数
④ 受众驱动分层同一篇要同时服务 业务/运营/测试/开发主体讲业务;warn 框放「开发/测试铁律」;可配参数表放运营
⑤ 现状对照每个功能/设计都标「现状 vs 待做」三色状态徽章 ✅已完成 / 🔵部分待验 / 🟡待完成,贯穿全库
⑥ 零构建可移植不打包、不跑服务器,双击即看纯 HTML 单文件 + 内联 CSS + 本地 Mermaid;不依赖 CDN/框架
⑦ 改图=改文本流程图不画图,用文本描述全部用 Mermaid;改流程就是改几行文本,diff 友好、AI 友好

2. 文档体系结构(分层)

一套文档分 6 层,职责单一、互相指向:

①  导航 hub        index.html              ← 所有入口 + 状态徽章 + 「先读这个」大卡片
②  业务总纲        overview.html           ← 一页读懂整个生意(必读,第一篇)
③  场景流程        scenario-N-flow.html    ← 用户旅程切片,每片一篇(Mermaid 流程图)
④  设计方案        *-design.html           ← 需要深入设计的模块(模型→流程→现状vs待做)
⑤  平台/架构/安全  architecture / auth …   ← 部署、鉴权、技术对齐
⑥  全盘状态        feature-status.html     ← 一张表看所有功能做到哪了
   参考/归档       assets/ _archive/ *.md  ← 本地 Mermaid+图、旧版归档、API/集成/QA 补充

BeeX 实例对照(22 个 HTML):

  • ① index.html
  • ② overview.html
  • ③ scenario-1..8-flow-v2.html(8 个场景:裂变/首购/升级/分佣/提现/机器人/券…)
  • ④ coupon-system-design.html、income-and-activities-design.html、campaign-placements-design.html
  • ⑤ beex-architecture.html、auth-design.html
  • ⑥ feature-status.html
  • 参考:assets/mermaid.min.js、_archive/(旧版)、api/*.md、prd/*.md、integrations/*.md、qa/*.md
关键:导航 hub(①)是唯一入口,团队只记一个地址,其它全从这里点进去。

3. 每类文档的「固定套路」(模板骨架)

③ 场景流程页(最高频,务必模板化)

← 返回文档导航
header(h1 标题 + sub 副标题)
<div class="intro">      一句话引子:这个场景在讲什么、给谁看
<h2 class="sec">一、主流程图
  <pre class="mermaid"> flowchart … </pre>     ← 核心:流程图
  <div class="legend">  图例(节点颜色含义)
<div class="warn">       🚨 铁律(开发/测试必看的硬规则,红框)
<div class="note">       📌 要点补充
<!-- CFG-PARAMS -->      ⚙️ 本场景可配参数(运营后台能调的:参数/初始值/配置位置/状态)
<footer>                 真值来源声明

④ 设计方案页

draft 状态条(🟡 设计稿/评审中)
intro(本稿覆盖哪几件事 + 依赖关系)
数据模型表(字段/口径)
流程(Mermaid 或分步)
「现状 vs 待做」表 + 分期(P1/P2/P3)
「真正残留」note(把没做完的单独拎出来,不混在已交付里)

② 业务总纲页(一页读懂)

固定回答 5 个问题(BeeX 的章节就是范本):

  1. 有哪几种人(角色 + 谁赚得多)
  2. 钱从哪来、怎么一层层分
  3. 怎么把盘子做大(裂变 + 升级)
  4. 一个人的完整旅程(从进来到做到顶)
  5. 接下来看哪(指向各场景)

⑥ 全盘状态表

summary 卡:✅N / 🔵N / 🟡N 计数
按域分表:功能 | 实现 | 状态徽章 | 备注/待办
缺口清单:按优先级(P0上线前/P1/P2/第二阶段)

① 导航 hub

hero:一句话原则声明(「不用任何黑话」「数值以…为准」)
lead-card:大卡片置顶「先读这个」(总纲 + 全盘状态表)
card grid:按域分组(裂变/核心业务/系统/设计/平台),每张卡带状态徽章
footer:真值来源

4. 统一规范(让 N 个散文件像「一套」)

规范做法
单文件自包含每个 .html 内联自己的 <style>,不依赖外部 CSS(file:// 友好)
Design token:root{ --honey/--ink/--bg … } 一处定义配色,全文件用变量;换项目只改 token
状态徽章三色✅ 已完成(绿 #15803d)/ 🔵 部分待验(蓝 #1d4ed8)/ 🟡 待完成(橙 #c2410c)
本地 Mermaid引 ./assets/mermaid.min.js(不用 CDN,file:// 也能渲染)
可配参数块每个场景嵌 <!-- CFG-PARAMS --> 表:参数/初始值/配置位置/状态——给运营
铁律框class="warn"(红)放开发/测试必看的硬规则
真值来源页脚每页 footer 声明数值/状态以什么为准,文档间不重复定义
返回导航每页顶部 ← 返回文档导航 链回 index
命名约定scenario-N-flow-vM.html / *-design.html / feature-status.html;旧版进 _archive/

通用 class 词表(全库统一):.sec(章节标题) .intro(引子框) .warn(铁律) .note(要点) .cmp(对比框) .cfg-h/.cfgtbl(可配参数) .card/.lead-card(导航卡) .tag.{biz,fis,sys,tech}(分类标) .tag.st-{done,wip,todo}(状态徽章)。

5. 为什么是「纯 HTML 单文件」(技术选型)

  • 零构建:不打包、不装 node、不跑 dev server,双击 file:// 即看——业务方/老板随时打开。
  • git 友好:纯文本,改动 diff 清楚。
  • AI 友好:单文件,AI 能整篇读/整篇改,不用拼上下文。
  • 图即文本:Mermaid 把流程图变成文本,改图=改几行字,版本可追溯。
  • 要对外就托管:需要发布时,整个目录扔静态托管(OSS/CDN/Pages)即可,不改一行。
  • 抗腐化:没有依赖版本、没有构建链,放三年打开还是这样。

6. 从 0 落地步骤(另一个项目照做)

  1. 先写 overview.html 总纲 —— 逼自己把整个业务一页讲清(讲不清说明还没想清)。
  2. 切场景 —— 把用户旅程切成几段,每段建一个 scenario-N-flow.html。
  3. 每场景三件事:① 画 Mermaid 主流程 ② 标 warn 铁律 ③ 列「本场景可配参数」。
  4. 抽设计方案页 —— 需要深入设计的模块单独成 *-design.html(模型→流程→现状vs待做)。
  5. 建 index.html 导航 hub —— 按域分组卡片 + 状态徽章 + 置顶「先读这个」。
  6. 建 feature-status.html —— 一张表看全盘现状 vs 待做。
  7. 定规范 —— 抄第 7 节的 token/class;定「禁用词表」+「真值来源」一句话。
  8. assets + _archive —— 本地 Mermaid 放 assets/;旧版别删,进 _archive/。
顺序很重要:总纲先行。先有一页读懂,再铺场景,最后建导航和状态表——而不是上来就建一堆空壳。

7.5 术语 hover 释义（bx-var glossary）

给任何设计稿 HTML 加「鼠标悬停术语弹窗」只需三件事：① CSS、② 占位 <div>、③ JS + G 对象（术语→释义）。JS 在页面加载时自动扫描全文包裹首次出现的每个术语，无需逐行手标。

CSS（加在 <style> 里，--brand-dark 换成文档的对应 token）

.bx-var{border-bottom:1px dashed var(--brand-dark,#e07c00);cursor:help;font-weight:600;}
.bx-tip{position:fixed;background:#1a1410;color:#fff;padding:11px 14px;border-radius:8px;
  font-size:12.5px;line-height:1.6;white-space:pre-line;max-width:360px;min-width:200px;
  z-index:10000;box-shadow:0 8px 24px rgba(0,0,0,.28),0 2px 6px rgba(0,0,0,.12);
  pointer-events:none;opacity:0;transform:translateY(4px);
  transition:opacity .13s ease-out,transform .13s ease-out;text-align:left;}
.bx-tip.on{opacity:1;transform:translateY(0);}
.bx-tip strong{color:var(--brand,#f9c74f);font-weight:700;}

HTML + JS（加在 </body> 之前，按文档替换 G 对象词条）

<div class="bx-tip" id="bx-tip"></div>
<script>
(function(){
  /* ① 术语→释义：第一行 = <strong> 标题，后续行 = 正文（\n 换行） */
  var G={
    'T1':'用户等级 T1 · 普通用户（User）',
    'T2':'用户等级 T2 · 小团长（Agent）',
    'T3':'用户等级 T3 · 大团长（Partner）',
    'bps':'bps · basis points\n1 bps = 0.01%，例：500 bps = 5%',
    // ← 在这里添加业务词……
  };
  var tip=document.getElementById('bx-tip');
  /* ② TreeWalker 自动包裹首次出现的术语（跳过 code/pre/script/mermaid） */
  var SKIP={SCRIPT:1,STYLE:1,CODE:1,PRE:1,A:1,FOOTER:1,H1:1,H2:1,H3:1};
  var used={},w=document.createTreeWalker(document.body,NodeFilter.SHOW_TEXT,null,false),q=[],nn;
  while(nn=w.nextNode())q.push(nn);
  q.forEach(function(tn){
    var p=tn.parentNode;if(!p)return;
    for(var cur=p;cur&&cur!==document.body;cur=cur.parentNode)
      if(SKIP[cur.tagName]||(cur.className&&/(mermaid|bx-var|bx-tip)/.test(cur.className)))return;
    var txt=tn.nodeValue;
    for(var k in G){if(used[k])continue;
      var i=txt.indexOf(k);if(i<0)continue;
      used[k]=1;var after=tn.splitText(i);after.splitText(k.length);
      var sp=document.createElement('span');
      sp.className='bx-var';sp.setAttribute('data-term',k);sp.textContent=k;
      after.parentNode.replaceChild(sp,after);break;}
  });
  /* ③ 鼠标 / 触摸事件 */
  function pos(el,t){var d=G[t];if(!d)return;
    var p=d.split('\n');
    tip.innerHTML='<strong>'+p[0]+'</strong>'+(p.length>1?'\n'+p.slice(1).join('\n'):'');
    tip.classList.add('on');
    var r=el.getBoundingClientRect(),tw=tip.offsetWidth||280,th=tip.offsetHeight||80;
    tip.style.left=Math.max(8,Math.min(r.left,window.innerWidth-tw-10))+'px';
    tip.style.top=((window.innerHeight-r.bottom>=th+20)?r.bottom+8:r.top-th-8)+'px';}
  document.addEventListener('mouseover',function(e){var el=e.target;for(;el&&el!==document.body;el=el.parentNode)if(el.className==='bx-var'){pos(el,el.getAttribute('data-term'));return;}});
  document.addEventListener('mouseout',function(e){var el=e.target;for(;el&&el!==document.body;el=el.parentNode)if(el.className==='bx-var'){tip.classList.remove('on');return;}});
  document.addEventListener('touchstart',function(e){var el=e.target;for(;el&&el!==document.body;el=el.parentNode)if(el.className==='bx-var'){pos(el,el.getAttribute('data-term'));return;}},{passive:true});
})();
</script>
心法：G 对象是单一真值来源——术语定义只维护一处，JS 自动扫全文包裹首次出现。换文档只改 G 里的词条；CSS 变量 --brand-dark 换成文档对应 token（如 --honey-dark、--gold）即可。

8. 一页速查(给另一个项目的人/AI)

想复刻这套?照这个清单走:
  1. 抄 §7.1 的 token+class、§7.2 的 Mermaid →（换配色)
  2. 先写 overview.html 一页读懂
  3. 切场景 → 每个 scenario-N.html:Mermaid + warn 铁律 + 可配参数表
  4. 深入模块写 *-design.html(模型→流程→现状vs待做)
  5. 建 index.html 导航(分组卡 + 状态徽章 + 置顶总纲)
  6. 建 feature-status.html 全盘表
  7. 定「禁用词表」+「真值来源一句话」,每页 footer 声明
  8. 本地 Mermaid 进 assets/,旧版进 _archive/,API/集成补充用 .md
心法:无黑话 · 一页读懂先行 · 单一真值来源 · 现状对照(三色徽章)· 零构建单文件 · 图即文本。
